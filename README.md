# smadimo_group_project_2

Бизнес задача - Мы аналитики компании, которая собирается ворваться на игровой рынок, у компании есть свой лаунчер и у нее есть ресурсы, чтобы быть конкурентноспобоной и выкупать игры, которые потенциально принесут выгоду.

Наша задача 

Проанализировать и отсделить перспективные жанры, и издателей, теги которые олицентворяют интересы потребителей и ОС/Платформы с которыми будем работать. Построить модель машинного обучения, которая прогнозирует рейтинг игроков и успешность игры 

Сбор данных через API

Наша предметная область - это видеоигры, поэтому как источник для апи мы выбрали RAWG IO

Краткое описание источника

RAWG дает большой доступ к открытой базе данных игр и предоставляет структурированные данные по более 500000 играм 
на более 50 различных платформ

Источник дает нам возможность брать метаданные такие как: название, дата выхода, жанры, разработчики и публикаторы
Так же платформа предоставляет нам информацию по таким метрикам как: агрегированные оценки Metacritic и внутренняя система оценок RAWG
Магазины: ссылки на площадки цифровой дистрибуции, где можно приобрести игру.

Технические детали и условия использования
Тип запросов: API работает по протоколу HTTP REST, возвращая данные в формате JSON.
Аутентификация: для доступа ко всем эндпоинтам требуется персональный API ключ

И самое важное и полезное - Документация: подробное описание всех методов и параметров доступно на официальной странице

Поля RAWG:

id - уникальный идентификатор игры в RAWG; по нему можно запрашивать карточку игры через /games/{id}.
name — официальное название игры. Это стандартное поле карточки игры и списка игр.
slug — текстовый идентификатор игры для URL и запросов; в RAWG id или slug могут использоваться как идентификатор игры.
released — дата релиза игры. RAWG прямо указывает, что API содержит release dates.
rating — агрегированный пользовательский рейтинг игры в RAWG.
ratings_count — число оценок, которые пользователи RAWG поставили игре.
reviews_count — число отзывов/ревью по игре в RAWG.
playtime — средняя длительность игры / среднее время игры
added — сколько пользователей RAWG добавили игру к себе в библиотеку/список.
suggestions_count — сколько раз игру “suggested” / рекомендовали / добавляли в рекомендации в экосистеме RAWG
genres — жанры игры. RAWG прямо перечисляет genres как одну из основных сущностей и частей metadata.
tags — теги игры. RAWG прямо перечисляет tags как часть rich metadata.
platforms — платформы, на которых вышла игра.
stores — магазины/цифровые площадки, где продаётся игра.
developers — разработчики игры. RAWG прямо перечисляет developers в составе metadata.
publishers — издатели игры. RAWG прямо перечисляет publishers в составе metadata.

Сбор данных через скрепинг

Краткое описание источника

Steam является крупнейшей платформой цифровой дистрибуции игр, где представлена информация о тысячах проектов
На страницах игр можно получить дополнительные данные, которых нет в RAWG

Источник дает нам возможность брать такие данные как: процент положительных отзывов, количество отзывов, текстовая оценка, цена игры и скидки
Также можно получить информацию о поддерживаемых платформах и уникальный идентификатор игры (steam_app_id), что важно для объединения датасетов

Технические детали и условия использования

Тип запросов: данные получаются через http запросы и парсинг html страниц
Используемые инструменты: requests и BeautifulSoup для извлечения данных из html

Дополнительно использовался Selenium, так как часть данных на сайте подгружается динамически и может быть недоступна при обычных запросах

Данные со Steam позволили дополнить API датасет реальными пользовательскими метриками и подготовить итоговый объединенный датасет для анализа

Поля для обогощения через Steam Store
steam_review_percent - процент положительных оценок
steam_review_count - кол-во оценок
steam_review_label - label, который словами описывает характер 
steam_price_text - цена игры
steam_discount_pct - процент скидки
steam_platform_flags - какие ОС поддерживается игрой
steam_app_id - стимовский айди
